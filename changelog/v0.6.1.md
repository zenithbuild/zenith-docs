# zenith-docs v0.6.1

## Changed

- **Tailwind contract:** Zenith now compiles Tailwind v4 internally from `src/styles/globals.css` when that file contains `@import "tailwindcss";` during both `zenith dev` and `zenith build`.
- **Dev CSS behavior:** canonical docs now point to `/__zenith_dev/styles.css` and `cssHref=/__zenith_dev/styles.css?buildId=<n>` as the only supported dev stylesheet contract.
- **Examples and starters:** docs now recommend a globals-only layout import pattern and remove `tw:build`, `output.css`, and `tailwind.css` guidance from canonical setup paths.

## Removed

- **Precompile requirement:** the old guidance that required a separate Tailwind CLI prebuild step before Zenith could serve CSS.

## Verification

- Packed-tarball smoke docs now match the internal Tailwind pipeline and the smoke gate validates that Zenith serves compiled CSS without any literal `tailwindcss` network request.
